# WebScout Agent — 웹사이트 발견부터 제품 실행까지

**단일 프로젝트로 Product Marketing, Design, AI-driven Product Building 역량을 동시에 증명하는 케이스 스터디**

---

## 개요

WebScout Agent는 웹사이트 분석과 디자인 실행 사이의 간극을 메우는 AI 기반 웹사이트 발견 에이전트입니다. 디자인 리빌드를 위한 URL 수집이라는 수동적이고 오류가 발생하기 쉬운 프로세스를 자동화된 지능형 워크플로우로 전환하여, 마케터와 디자이너 모두에게 실행 가능한 인사이트를 제공합니다.

**핵심 가치**: 웹사이트 → 구조 분석 → 디자인 실행 — 모두 하나의 AI 에이전트로 연결됩니다.

URL을 하나씩 수동으로 입력하는 데 몇 시간을 소비하는 대신, WebScout Agent는 자동으로 웹사이트 URL을 발견하고, 정규화하고, 구조화하고, Figma 임포트에 바로 사용할 수 있도록 내보내면서, 페이지 타입, 중요도, 디자인 준비도에 대한 인텔리전스를 제공합니다.

## 왜 만들었는가

### 문제점

웹사이트 디자인을 리빌드할 때 세 가지 중요한 문제점이 발생합니다:

1. **수동 URL 수집은 비효율적**
   - 디자이너와 마케터가 URL을 수집하는 데 몇 시간을 소비
   - 높은 오류율과 누락된 페이지
   - 완전성을 보장하는 체계적인 접근 방법 부재

2. **구조 이해가 어려움**
   - 웹사이트 계층 구조를 시각화하기 어려움
   - 어떤 페이지가 가장 중요한지 불명확
   - 페이지 간 관계를 자동으로 이해하는 방법 부재

3. **분석과 제작이 분리됨**
   - 웹사이트 분석 도구가 디자인 도구와 연결되지 않음
   - 분석과 Figma 간 수동 데이터 전송
   - 인수인계 과정에서 컨텍스트 손실

### 기회

웹사이트 발견부터 Figma-ready 내보내기까지 전체 플로우를 자동화하면서, 마케팅 전략과 디자인 결정을 모두 돕는 인텔리전스를 제공할 수 있다면 어떨까요?

## WebScout Agent가 하는 일

### 에이전트 플로우

```
도메인 입력
    ↓
Sitemap 발견 (robots.txt → sitemap.xml)
    ↓
크롤링 대체 (sitemap이 없는 경우)
    ↓
URL 정규화 (www/non-www 병합, 쿼리 제거, 해시 제거)
    ↓
구조 분석 (깊이 계산, 경로 계층 구조)
    ↓
인텔리전스 레이어 (페이지 타입 태깅, 링크 관계 매핑)
    ↓
인사이트 생성 (Figma 준비도 점수, 중요도 히트맵)
    ↓
Figma-Ready 내보내기 (urls.txt / urls.csv)
```

### 핵심 기능

**발견**
- sitemap.xml 자동 감지 및 파싱
- 지능형 내부 링크 크롤링으로 대체
- robots.txt 및 same-origin 정책 준수

**정규화**
- www와 non-www 변형 병합
- 쿼리 파라미터 제거 (선택사항)
- 해시 프래그먼트 제거
- 제외 경로 필터링

**인텔리전스**
- 계층 구조 이해를 위한 경로 깊이 계산
- 중요도 점수를 위한 링크 관계 추적
- 페이지 타입 자동 태깅 (marketing/product/support/other)
- URL 품질 기반 Figma 준비도 점수

**내보내기**
- urls.txt 생성 (한 줄에 하나, html.to.design 호환)
- 메타데이터가 포함된 urls.csv 생성
- 선택적 내보내기 지원

## 마케팅 관점

### 퍼널 사고

WebScout Agent는 마케터가 전환 퍼널 관점에서 웹사이트 구조를 빠르게 이해할 수 있게 합니다:

**랜딩 → 제품 → 전환**

Sitemap과 페이지 타입을 분석함으로써 마케터는:
- 랜딩 페이지와 전환 경로 식별
- 콘텐츠 구조와 계층 구조 이해
- 웹사이트를 통한 사용자 여정 매핑
- 퍼널의 공백 발견

**사용 사례 예시**: 경쟁사 도메인 입력 → Sitemap 구조 분석 → 콘텐츠 전략 이해 → 차별화 기회 식별

### 경쟁사 분석

경쟁사 웹사이트를 분석할 때:

1. **경쟁사 도메인 입력** → WebScout Agent가 모든 URL 발견
2. **인사이트 검토** → 페이지 타입이 콘텐츠 전략을 드러냄
3. **중요도 분석** → 링크 수가 트래픽을 유도하는 페이지를 보여줌
4. **분석용 내보내기** → 스프레드시트에서 추가 분석을 위해 CSV 사용

이것은 몇 시간의 수동 연구를 몇 분의 자동화된 발견으로 전환합니다.

### 실행 워크플로우

**마케팅 → 디자인 → 런칭**

```
경쟁사 분석 (WebScout Agent)
    ↓
콘텐츠 전략 인사이트
    ↓
Figma 디자인 (URL 임포트)
    ↓
랜딩 페이지 제작
    ↓
A/B 테스트 설정
```

WebScout Agent는 이 워크플로우의 시작점에 위치하여 데이터 기반 마케팅 결정의 기반을 제공합니다.

## 디자인 관점

### 정보 아키텍처

**자동화된 구조 발견**

WebScout Agent는 자동으로 생성합니다:
- **Sitemap 트리 뷰** - 모든 페이지의 시각적 계층 구조
- **경로 깊이 분석** - 사이트 구조 깊이 이해
- **페이지 관계** - 어떤 페이지가 어떤 페이지로 링크되는지 (중요도 매핑)

이것은 수동 sitemap 분석을 대체하고 디자이너에게 다음을 제공합니다:
- 한눈에 보는 완전한 사이트 구조
- 페이지 중요도 이해
- 네비게이션 디자인의 기반

### UX 디자인

**디자인 시스템 구현**

WebScout Agent 대시보드는 다음을 보여줍니다:

**히어로 섹션**
- 명확한 가치 제안
- 주요 CTA (Collect URLs)
- 플로우 표시기 (Discover → Normalize → Structure → Export)

**필터 바**
- 깊이, 최대 URL, 제외에 대한 직관적인 컨트롤
- 시각적 칩 기반 인터페이스
- 반응형 디자인

**KPI 대시보드**
- 실시간 메트릭 (총 URL 수, 출처, 깊이, 제외된 수)
- 액센트 카드가 있는 시각적 계층 구조
- 깔끔하고 스캔 가능한 레이아웃

**인터랙티브 테이블**
- 검색 기능
- 정렬 가능한 컬럼
- 페이지네이션
- 일괄 선택

**인사이트 탭**
- Sitemap 트리 시각화
- 페이지 타입 분류
- Figma 준비도 점수
- 중요도 히트맵

**적용된 디자인 원칙**:
- 화이트 기반 SaaS 미학
- 일관된 간격과 타이포그래피
- 미묘한 애니메이션과 마이크로 인터랙션
- 모바일 반응형 레이아웃
- 접근성 고려사항

### 디자인 준비도

**Figma-Ready 휴리스틱**

WebScout Agent는 각 URL에 대해 Figma 임포트 준비도를 점수화합니다:

- **깔끔한 URL** (+10) - 쿼리 파라미터 없음
- **짧은 경로** (+10) - 간단하고 기억하기 쉬운 URL
- **해시 없음** (+5) - 깔끔한 프래그먼트
- **메타데이터 사용 가능** (+15) - 디자인을 위한 풍부한 컨텍스트
- **일반적인 패턴** (+10) - 표준 URL 구조

이것은 디자이너가 어떤 페이지를 먼저 디자인할지 우선순위를 정하는 데 도움이 되며, 깔끔하고 프로덕션 준비된 URL을 보장합니다.

## 제품 사고

### 문제 정의

**사용자 문제**: 수동 URL 수집은 느리고 오류가 발생하기 쉬우며 인사이트를 제공하지 않습니다.

**비즈니스 문제**: 디자인 리빌드가 너무 오래 걸려 시장 출시 시간을 지연시킵니다.

**해결책**: 하나의 도구에서 자동화된 발견 + 인텔리전스 + 내보내기

### 사용자 플로우

```
1. 마케터/디자이너가 웹사이트를 리빌드해야 함
   ↓
2. WebScout Agent에 도메인 입력
   ↓
3. 실시간 진행 상황 및 KPI 메트릭 검토
   ↓
4. 인사이트 탐색 (구조, 타입, 준비도)
   ↓
5. Figma 임포트를 위해 urls.txt 내보내기
   ↓
6. html.to.design에 임포트
   ↓
7. 완전한 URL 목록으로 디자인 시작
```

### MVP 설계

**1단계: 핵심 기능**
- URL 발견 (sitemap + crawl)
- 기본 정규화
- 간단한 내보내기

**2단계: 인텔리전스 레이어**
- 페이지 타입 태깅
- 중요도 점수화
- Figma 준비도

**3단계: UI 폴리싱**
- SaaS급 대시보드
- 인사이트 탭
- 반응형 디자인

**반복적 개선**: 마이크로 디자인 패스, 타이포그래피 정제, 모션 디자인을 통해 9.3에서 9.9 SaaS 품질로 향상

### SaaS 구조

**아키텍처 결정**:
- **서버리스 우선** - 인프라 제로를 위한 Vercel 배포
- **데이터베이스 없음** - 상태 없는 파일 기반 출력
- **API 설계** - 실시간 업데이트를 위한 SSE가 있는 RESTful 엔드포인트
- **확장성** - 수천 개의 URL을 효율적으로 처리

**제품 결정**:
- **무료 티어** - 인증 불필요
- **셀프 호스팅 옵션** - 오픈 소스, 로컬에서 실행 가능
- **내보내기 형식** - 다양한 사용 사례를 위한 여러 형식

## 기술 + AI 레이어

### 기술 스택

**백엔드**
- ES 모듈이 있는 Node.js 18+
- API 및 정적 제공을 위한 Express.js
- 서버리스 함수 아키텍처 (Vercel)

**프론트엔드**
- 바닐라 JavaScript (프레임워크 오버헤드 없음)
- 커스텀 속성이 있는 모던 CSS
- 모바일 우선 접근 방식의 반응형 디자인

**핵심 로직**
- Sitemap XML 파싱
- HTML 링크 추출
- URL 정규화 알고리즘
- 그래프 기반 링크 관계 추적

### AI 기반 기능

**규칙 기반 인텔리전스**

머신러닝 모델을 사용하지 않지만, WebScout Agent는 지능적인 규칙을 적용합니다:

1. **페이지 타입 분류**
   - URL 경로의 패턴 매칭
   - 키워드 기반 분류
   - 컨텍스트 인식 태깅

2. **중요도 점수화**
   - 링크 수 분석
   - 깊이 기반 가중치
   - 관계 매핑

3. **Figma 준비도 휴리스틱**
   - 다중 요소 점수화
   - 품질 지표
   - 프로덕션 준비도 평가

**규칙 기반을 선택한 이유?**
- 빠르고 결정론적
- 설명 가능한 결과
- 훈련 데이터 불필요
- 모든 웹사이트에서 즉시 작동

### 인사이트 대시보드

**네 가지 인텔리전스 탭**:

1. **Sitemap View** - 웹사이트 구조의 트리 시각화
2. **Page Types** - 자동 분류 (marketing/product/support/other)
3. **Figma Ready** - 디자인 임포트를 위한 준비도 점수
4. **Importance** - 링크 관계 기반 히트맵

이러한 인사이트는 원시 URL 수집을 실행 가능한 인텔리전스로 전환합니다.

1. **인텔리전스 레이어** - 단순 수집이 아닌 인사이트
2. **디자인 통합** - Figma 워크플로우를 위해 구축
3. **마케팅 관점** - 퍼널 및 경쟁사 분석
4. **SaaS 품질** - 프로덕션 준비, 프로토타입 아님
5. **오픈 소스** - 투명하고 커스터마이징 가능하며 무료

## 요약

WebScout Agent는 다음의 교차점을 나타냅니다:

**마케팅 + 디자인 + 제품 + 자동화**

단일 프로젝트를 통해 다음을 증명합니다:

- **마케팅 전략** - 퍼널 사고, 경쟁사 분석, 워크플로우 자동화
- **디자인 우수성** - 정보 아키텍처, UX 디자인, 디자인 시스템
- **제품 사고** - 문제 정의, 사용자 플로우, MVP 실행
- **AI 역량** - 지능형 자동화, 규칙 기반 시스템, 인사이트 생성

이것은 단순한 도구가 아닙니다—**실제 문제를 해결하는 제품을 구축하는 동시에 다기능 역량을 증명하는 케이스 스터디**입니다.

**핵심 요약**: 제품 사고, 디자인 우수성, 마케팅 인식을 접근 방식으로 사용할 때, 하나의 프로젝트가 여러 기술 세트를 보여줄 수 있습니다.
