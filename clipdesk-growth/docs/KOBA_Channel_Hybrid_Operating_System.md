# 4. KOBA 전시 실행 플랜

## Phase 1: 브랜드 정렬 (Week 1)

- 메시지 1줄 확정
- 3사업부 Use Case 1줄 각각 확정
- KPI 시트 공유

## Phase 2: 콘텐츠 준비 (Week 2–3)

- 랜딩·사전 등록 페이지
- 부스 비주얼·인쇄물
- 데모 샘플 (Edit / Transcription / Caption)
- 초대 이메일 초안

## Phase 3: 사전 홍보 (Week 3–5)

- LinkedIn, Threads, 블로그 KOBA 콘텐츠
- 초대 이메일 1·2차 발송
- 사전 미팅 목표 10–20건

## Phase 4: 현장 운영

### 현장 역할표

| 역할 | 담당 | 시간대별 | 할 일 |
|------|------|----------|------|
| 접객·안내 | Edit / Caption | 교대 | QR 스캔 유도, 명함 수집 |
| 데모 | 해당 사업부 | 교대 | Edit / Transcription / Caption 데모 |
| 리드 정리 | Transcription | 퇴장 후 | 당일 리드 이관 |
| 총괄 | Growth Owner | 전체 | 동선·이슈 대응 |

### 현장 체크리스트

- [ ] QR 폼 테스트
- [ ] 명함·펜·물티슈 비치
- [ ] 데모 환경(노트북·모니터) 점검
- [ ] 리드 수집 시트/태블릿 준비

## Phase 5: 48시간 Follow-up

| 시점 | 담당 | 할 일 |
|------|------|------|
| D-day 종료 | Transcription | 리드 CRM/엑셀 이관 |
| D+1 | Growth Owner | 우선순위 분류 |
| D+2 내 | Edit / Caption / Transcription | 감사 이메일 + 1차 전화 100% |

# 5. 리드 수집 프로세스

## 플로우

```
부스 QR 스캔 / 명함 / 태블릿 입력
    ↓
리드 폼 제출
    ↓
Webhook → DB (Notion / Supabase 등)
    ↓
Slack 알림 → 해당 사업부
    ↓
48시간 내 감사 메일 + 1차 전화
```

## 폼 필드 정의

| 필드 | 타입 | 필수 | 용도 |
|------|------|------|------|
| name | text | Y | 이름 |
| company | text | Y | 회사명 |
| email | email | Y | 이메일 |
| phone | tel | N | 전화 |
| interest | select | Y | caption / transcription / edit / 기타 |
| meeting | select | Y | Y / N |
| source | hidden | - | koba |
| utm_source | hidden | - | koba |

## CTA

- 제출 버튼: "상담 신청"
- 제출 후: 감사 페이지 or 자동 응답 이메일
