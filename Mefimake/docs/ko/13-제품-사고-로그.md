# 제품 사고 로그 – 크리에이티브 툴에서 자동화 시스템까지

> 이 문서는 MEFIMAKE의 실제 제품 사고 과정, 전략적 전환, 워크플로우 진화를 담고 있습니다. 요약이나 마케팅 자료가 아닙니다. 빌더의 저널입니다.

---

## 1. 기원: 왜 이 도구를 만들었는가

### 이것은 스타트업 아이디어가 아니었다

MEFIMAKE는 피치덱으로 시작하지 않았다. 시장 조사 단계도 없었다. 경쟁사 분석 스프레드시트도 없었다.

좌절에서 시작했다.

### 실제 운영의 고통

임원들에게 AI 자동화를 설명해야 하는 역할에서, 반복되는 문제에 직면했다:

**사람들은 경험하지 못한 것을 이해하지 못한다.**

매주:
- "AI 역량"에 대한 또 다른 프레젠테이션
- "프롬프트 엔지니어링"에 대한 또 다른 교육 세션
- "워크플로우 자동화"에 대한 또 다른 개념적 설명

그리고 매주:
- 멍한 눈빛
- 예의 바른 끄덕임
- 제로 채택

임원들이 멍청한 게 아니었다. 바빴다. AI 워크플로우가 무엇을 할 수 있는지 상상할 시간이 없었다. 그들은 느껴야 했다.

### 구체적인 고통들

1. **반복적인 크리에이티브 작업**
   - 같은 광고 포맷, 다른 키워드
   - 같은 레이아웃, 다른 메시지
   - 몇 분이면 될 일에 수 시간 소비

2. **이해관계자에게 AI 설명하기**
   - "AI가 X에 도움이 될 수 있어요" → "보여줘봐"
   - 시연이 실제 작업보다 오래 걸림
   - 회의 후 이해가 증발

3. **수동 경쟁사 분석**
   - 경쟁사 랜딩 페이지 스크롤링
   - 섹션 스크린샷
   - 메시징 패턴 수동 문서화
   - 체계적 캡처 없음

4. **느린 콘텐츠 제작**
   - 브리프 → 디자이너 → 검토 → 수정 → 내보내기
   - 너무 많은 핸드오프
   - 너무 많은 대기

### 핵심 통찰

> "이것은 임원들이 설명을 듣는 대신 AI 워크플로우를 직접 경험할 수 있도록 만들어졌다."

자동화를 설명하려는 시도를 멈췄다.
그들이 만질 수 있는 무언가를 만들었다.

## 2. 전략 전환: 설명에서 경험으로

### 전환점

수개월간 수익이 체감하는 프레젠테이션 후, 깨달았다:

**교육은 확장되지 않는다. 도구는 확장된다.**

대신에:
- AI 잠재력에 대한 파워포인트 덱
- 프롬프트 작성에 대한 워크숍 세션
- 자동화에 대한 이론적 토론

접근 방식이 바뀌었다:

**도구를 줘라. 느끼게 해라.**

### 내부 데모 마인드셋

이것은 사용자를 위해 만들어지지 않았다. 이해관계자를 위해 만들어졌다.

대상은 "퍼포먼스 마케터"가 아니었다. 대상은:
- 예산을 승인하는 부서장
- 리소스를 배정하는 팀 리드
- 믿어야 하는 의사결정자

### 첫 사용 경험의 중요성

누군가가 이 도구를 열고 3분 안에 "이해"하지 못하면, 영원히 닫는다.

**"첫 3분이 중요하다" 원칙:**

1. **0-1분:** 익숙한 것을 본다 (광고 프리뷰)
2. **1-2분:** 무언가를 바꾸고 결과를 본다 (텍스트 편집 → 라이브 업데이트)
3. **2-3분:** 새로운 것을 생성한다 (AI 콘텐츠 → 내보내기)

3분 안에 진전을 느끼지 못하면, 떠난다.

이것이 모든 것을 형성했다:
- 온보딩 튜토리얼 없음
- 빈 상태 없음
- 기본 콘텐츠가 이미 로드됨
- 첫 출력까지 원클릭

### 베팅

경험이 설명보다 더 잘 전환될 것이라고 베팅했다.

효과가 있었다.

MEFIMAKE를 5분간 사용한 후, 임원들이 묻기 시작했다: "다른 건 뭘 자동화할 수 있어?"

그 질문은 프레젠테이션에서는 절대 나오지 않았다.

## 3. 워크플로우 아키텍처 (실제 시스템)

### 전체 파이프라인

MEFIMAKE는 독립형 도구가 아니다. 더 큰 시스템의 한 노드다.

```
시그널 → 구조 → 생성 → 자동화
```

### 구성 요소

#### 1. RSS + 키워드 모니터링 (시그널 감지)

**역할:** 경쟁사보다 먼저 시장 움직임 감지.

- 산업 뉴스 피드
- 경쟁사 블로그 모니터링
- 키워드 트렌드 추적
- 소셜 시그널 집계

**출력:** 무언가 일어나고 있다는 원시 시그널.

#### 2. Notion 데이터베이스 (인사이트 저장소)

**역할:** 시그널을 구조화된 인사이트로 변환.

- 주제, 긴급성, 기회별 분류
- 관련 캠페인 태그
- 소스 자료 링크
- 크리에이티브 브리프용 쿼리 가능

**출력:** 크리에이티브 결정을 위한 정리된 지식 베이스.

#### 3. WebScout Agent (경쟁 구조 분석)

**역할:** 경쟁사가 무엇을 말하는지가 아니라 어떻게 소통하는지 이해.

- 랜딩 페이지 구조 추출
- 섹션 계층 매핑
- 메시징 패턴 식별
- CTA 배치 분석

**출력:** 크리에이티브 아이디어를 위한 구조적 템플릿.

#### 4. MEFIMAKE (크리에이티브 생성)

**역할:** 인사이트로부터 시각적 광고 크리에이티브를 빠르게 제작.

- 텍스트 우선 디자인 접근
- 멀티 사이즈 출력
- 변형 생성
- 내보내기 준비 에셋

**출력:** 제작 준비된 광고 크리에이티브.

#### 5. Make (향후 자동화 레이어)

**역할:** 모든 조각을 자율 워크플로우로 연결.

- MEFIMAKE 내보내기 → Notion 로깅
- Notion 태스크 → Make 트리거
- Make 워크플로우 → 배포 채널

**출력:** 자체 실행 크리에이티브 운영.

### 핵심 통찰

> 이것은 여러 도구가 아니다. 이것은 하나의 연속된 워크플로우다.

각 구성 요소는 특정 갭을 채우기 때문에 만들어졌다 (또는 선택되었다):

| 갭 | 솔루션 |
|-----|------|
| "시장에서 무슨 일이 일어나는지 모르겠어" | RSS 모니터링 |
| "지난주에 본 인사이트를 찾을 수 없어" | Notion 데이터베이스 |
| "경쟁사가 페이지를 어떻게 구성하는지 모르겠어" | WebScout Agent |
| "크리에이티브를 충분히 빨리 제작할 수 없어" | MEFIMAKE |
| "아직도 수동으로 하고 있어" | Make 자동화 |

목표는 "도구"를 만드는 게 아니었다.
목표는 "시스템"을 만드는 것이었다.

## 4. WebScout Agent가 존재하는 이유

### 좌절

모든 경쟁사 분석은 같은 방식으로 시작했다:

1. 경쟁사 웹사이트 열기
2. 랜딩 페이지 스크롤
3. 흥미로운 섹션 스크린샷
4. 구글 독에 붙여넣기
5. 발견한 것에 대해 노트 작성
6. 10개 경쟁사에 대해 반복
7. 스크린샷 절반을 저장 안 했다는 걸 깨달음
8. 처음부터 다시 시작

이것은 수 시간이 걸렸다. 그리고 출력물은 정리되지 않았다.

### 실제로 필요했던 것

스크린샷이 아니라. **구조.**

- 랜딩 페이지에 몇 개의 섹션이 있는가?
- 헤드라인 계층은 어떤가?
- 추천사를 어디에 배치하는가?
- CTA 진행은 어떤가?
- 어떤 메시징 패턴이 반복되는가?

### WebScout Agent가 하는 일

웹페이지의 뼈대를 추출한다:

```
랜딩 페이지 분석:
├── 히어로 섹션
│   ├── 헤드라인: "비즈니스를 변환하세요"
│   ├── 서브헤드: "AI 기반 솔루션..."
│   └── CTA: "무료 체험 시작"
├── 소셜 프루프 섹션
│   ├── 로고 바: 6개 회사 로고
│   └── 추천사 수: 3
├── 기능 섹션
│   ├── 3열 레이아웃
│   └── 아이콘 + 헤딩 + 설명 패턴
...
```

이것은 크리에이티브 아이디어를 직접 피드한다. 막연한 영감 대신, 구조적 템플릿을 얻는다.

### 시스템에서의 위치

> WebScout Agent는 시스템의 **Discover 레이어**다.

질문에 답한다: "다른 사람들은 무엇이 효과적인가?"

## 5. MEFIMAKE가 존재하는 이유

### 제거하는 마찰

전통적인 크리에이티브 워크플로우:

```
아이디어 → 브리프 → 디자이너 → 검토 → 피드백 → 수정 → 내보내기
              ↑__________________________________________|
                         (3-5회 반복)
```

MEFIMAKE 워크플로우:

```
아이디어 → MEFIMAKE → 내보내기
```

한 사람. 한 도구. 한 세션.

### 디자인 철학: 텍스트 + 컬러 우선

대부분의 크리에이티브 도구는 다음으로 시작한다:
- 캔버스
- 도형
- 이미지
- 효과

MEFIMAKE는 다음으로 시작한다:
- 무엇을 말하고 싶은가?
- 어떤 색이 브랜드를 나타내는가?

이미지는 선택 사항이다. 효과는 최소화. 초점은 메시지다.

### 왜 이것이 중요한가

광고는 장식이 아니라 메시징으로 성공하거나 실패한다.

내가 분석한 가장 효과적인 Meta 광고들:
- 단색 배경
- 볼드 텍스트
- 명확한 CTA
- 화려한 이미지 없음

MEFIMAKE는 이 현실을 위해 만들어졌다.

### 이것이 아닌 것

> "이것은 Canva가 아니다. 이것은 사고 인터페이스다."

Canva는 예쁘게 만들기 위한 것이다.
MEFIMAKE는 변형을 통해 사고하기 위한 것이다.

최종 픽셀에 관한 게 아니다. 반복 속도에 관한 것이다.

## 6. 철학: 인간 + AI 협업

### 완전 자동화가 아니다

일부 프로세스는 엔드 투 엔드로 자동화되어야 한다.
크리에이티브 제작은 그중 하나가 아니다.

왜? 크리에이티브 결정은 AI가 복제할 수 없는 판단을 필요로 하기 때문이다:
- 이 메시지가 우리 브랜드에 맞는 느낌인가?
- 이게 우리 오디언스에게 너무 공격적인가?
- 이게 캠페인 목표와 일치하는가?

### AI의 역할

AI는 가속하지만 대체하지 않는다:

| 태스크 | 인간 역할 | AI 역할 |
|--------|----------|---------|
| 아이디어 발상 | 방향 | 생성 |
| 카피 | 승인 | 변형 |
| 디자인 | 취향 | 실행 |
| 내보내기 | 선택 | 배치 처리 |

### 왜 이미지 업로드 마찰이 허용되는가

MEFIMAKE는 내장 이미지 라이브러리가 없다. 사용자가 직접 업로드해야 한다.

이것은 의도적이다.

**이유:**
1. 이미지 선택은 브랜드 판단을 필요로 함
2. 스톡 이미지는 종종 브랜드 보이스와 맞지 않음
3. 업로드 강제가 이미지에 대해 생각하게 함
4. "편한 거 아무거나 가져오기" 게으름 방지

### 왜 텍스트 우선 디자인이 중요한가

이미지로 시작하면 메시지가 제약된다.
메시지로 시작하면 디자인이 자유로워진다.

"이번 주말 50% 할인"을 먼저 쓰면, 디자인이 메시지를 섬긴다.
해변 사진을 먼저 고르면, 메시지가 디자인을 섬긴다.

MEFIMAKE는 올바른 순서를 강제한다.

### 왜 부분적 수동 단계가 의도적인가

완전 자동화는 학습을 제거한다.

누군가가 수동으로 "생성"을 클릭하고 5개 변형이 나타나는 것을 보면, 무슨 일이 일어났는지 이해한다.

자동으로 일어났다면, 마법일 것이다—인상적이지만 이해할 수 없는.

목표는 감탄이 아니라 채택이다.

## 7. 제품 진화 하이라이트

### 선택 시스템 리디자인

**문제:** 사용자들이 배경 요소를 클릭하고 레이아웃을 망가뜨림.

**반복:**
- 첫 번째: 모든 요소 선택 가능 → 혼란
- 두 번째: "잠금" 기능 추가 → 사용자가 사용을 잊음
- 세 번째: 배경 격리 규칙 → 배경은 절대 선택 불가

**결과:** 실수로 배경 선택하는 경우 제로.

**교훈:** 사용자에게 해가 되는 옵션을 주지 말라.

---

### 레이어 바인딩 변경

**문제:** 레이어 패널과 캔버스가 동기화되지 않음.

**반복:**
- 첫 번째: 수동 동기화 → 지속적인 버그
- 두 번째: 단방향 바인딩 → 선택 불일치
- 세 번째: 양방향 동기화 → 작동

**결과:** 캔버스에서 선택 = 패널에서 하이라이트. 패널에서 선택 = 캔버스에서 선택.

**교훈:** 두 가지가 항상 일치해야 한다면, 코드로 강제하라.

---

### 배경 격리

**문제:** 배경이 선택됐을 때 프로퍼티 패널이 잘못된 속성을 표시.

**반복:**
- 첫 번째: 프로퍼티 패널에서 요소 타입 체크 → 너무 많은 조건문
- 두 번째: 배경 선택 자체를 방지 → 깔끔

**결과:** 배경은 전용 컨트롤을 가지고, 선택 플로우에 절대 들어가지 않음.

**교훈:** 엣지 케이스를 방지하여 제거하라.

---

### 프로퍼티 파이프라인 통일

**문제:** 폰트 사이즈 입력, 컬러 피커, 위치 입력이 모두 다르게 작동.

**반복:**
- 첫 번째: 각 입력 타입이 커스텀 로직 → 유지보수 악몽
- 두 번째: 통일된 패턴: 입력 → 핸들러 → 스타일 → 상태 → 저장

**결과:** 모든 프로퍼티 변경이 같은 파이프라인을 통과.

**교훈:** 지루한 일관성이 영리한 변형을 이긴다.

---

### 컬러 깊이 고민

**문제:** UI가 평면적으로 보임. 시각적 계층 없음.

**반복:**
- 첫 번째: 단일 어두운 색 → 모든 것이 섞임
- 두 번째: 3단계 계층 → 너무 미묘함
- 세 번째: iOS Dark Mode 참조 → 명확한 레벨
- 네 번째: 사용자 정의 팔레트 → 기대와 일치

**결과:** 다크 테마를 깨지 않으면서 뚜렷한 시각적 레이어.

**교훈:** 추상적 원칙이 아닌 실제 제품을 참조하라.

---

### 인트로 리디자인

**문제:** 여러 인트로 디자인이 "이상하게" 느껴짐.

**반복:**
- 첫 번째: 3D 큐브와 모션 → 너무 복잡
- 두 번째: 글래스모피즘 → 여전히 이상함
- 세 번째: 단순한 텍스트 + 로고 → 깔끔

**결과:** "Build high-converting creatives with AI." + 로고 + 로더.

**교훈:** 모든 것이 잘못 느껴질 때, 맞게 느껴질 때까지 제거하라.

---

### 온보딩 플로우 변경

**문제:** 사용자가 Square (1:1) 대신 Story (9:16) 사이즈에 도착.

**반복:**
- 첫 번째: 저장된 상태가 로드 시 복원 → 잘못된 기본값
- 두 번째: 초기 상태를 Square로 강제 → 올바른 경험

**결과:** 모든 새 세션이 Square 선택으로 시작.

**교훈:** 기본 상태가 가장 중요한 상태다.

## 8. 감정적 & 인지적 전환

### 깨달음

이 과정 어딘가에서, 무언가를 알아차렸다:

> "나는 페이지를 디자인하는 것보다 시스템을 디자인하는 것을 즐긴다."

랜딩 페이지 만들기: 만족스럽지만 유한하다.
워크플로우 시스템 만들기: 끝없지만 에너지를 준다.

### 전환

**이전:**
- "이 페이지를 어떻게 더 보기 좋게 만들까?"
- "이 기능을 어떻게 작동하게 하지?"
- "이 프로젝트를 어떻게 끝내지?"

**이후:**
- "이 조각들이 어떻게 연결되지?"
- "이 플로우에서 마찰이 어디 있지?"
- "무엇이 자동으로 vs. 수동으로 일어나지?"

### 페이지에서 플로우로

| 이전 사고 | 새로운 사고 |
|----------|------------|
| 홈페이지 만들기 | 생성 플로우 만들기 |
| 기능 추가하기 | 워크플로우 단계 추가하기 |
| 도구 출시하기 | 시스템 출시하기 |

### 커리어 전환 인사이트

이 프로젝트는 내가 실제로 무엇을 하고 싶은지 드러냈다:

아님: 것들을 만들기
맞음: 것들을 만드는 시스템을 만들기

아님: 인터페이스를 디자인하기
맞음: 워크플로우를 디자인하기

아님: 제품을 만들기
맞음: 자동화 아키텍처를 만들기

이 로그는 그 전환의 증거다.

## 9. 내부 임팩트 목표

### 의도된 효과

MEFIMAKE는 출력물을 생산하는 게 아니라 마인드를 바꾸기 위해 만들어졌다.

**타겟: 마케팅 팀**
- 이전: "AI는 IT가 처리하는 거야"
- 이후: "방금 2분 만에 15개 광고 변형을 만들었어"

**타겟: 기획 팀**
- 이전: "자동화는 비싸게 들린다"
- 이후: "이게 우리 워크플로우에 어떻게 연결되는지 보여"

**타겟: 리더십**
- 이전: "AI 투자의 ROI를 보여줘"
- 이후: "이것처럼 다른 건 뭘 자동화할 수 있어?"

### 이 도구는 촉매제다

> 최종 제품이 아니다. 내부 역량을 위한 개념 증명이다.

성공 지표는 DAU나 수익이 아니다.
성공 지표는: **이해관계자들이 다른 질문을 하기 시작했는가?**

## 10. 향후 자동화 레이어 (Make)

### 현재 상태: 수동 인텔리전스

지금, 시스템은 이렇게 작동한다:

1. 인간이 시그널을 모니터링 (RSS, 소셜)
2. 인간이 인사이트를 저장 (Notion)
3. 인간이 경쟁사를 분석 (WebScout)
4. 인간이 콘텐츠를 생성 (MEFIMAKE)
5. 인간이 배포 (수동 업로드)

인텔리전스는 높다. 자동화는 낮다.

### 다음 상태: 자동화된 실행

목표:

1. 자동화된 시그널 모니터링 (RSS → Notion)
2. 자동화된 인사이트 태깅 (Notion AI)
3. 온디맨드 경쟁 분석 (WebScout API)
4. 반자동화된 크리에이티브 생성 (MEFIMAKE + Make)
5. 자동화된 배포 (Make → 플랫폼)

### 연결 레이어

```
MEFIMAKE → Notion
Notion → Make
Make → 배포 채널
```

**현재 단계:** 수동 인텔리전스
**다음 단계:** 자동화된 실행

### 왜 지금 자동화하지 않는가?

조급한 자동화는 나쁜 프로세스를 고정시키기 때문이다.

첫째: 올바를 때까지 수동으로 하라.
그 다음: 검증된 것을 자동화하라.

우리는 "수동으로 하기" 단계에 있다. 의도적으로.

## 11. 제품 정체성

### MEFIMAKE가 아닌 것

**디자인 도구가 아니다.**
- 벡터 편집 없음
- 픽셀 조작 없음
- 디자인 시스템 기능 없음

**광고 생성기가 아니다.**
- 광고 계정 연결 없음
- 성과 추적 없음
- 입찰 최적화 없음

### MEFIMAKE가 무엇인가

> AI 크리에이티브 워크플로우 시스템.

더 큰 자동화 파이프라인의 "생성" 단계다.

```
발견 → 결정 → 생성 → 배포
              ↑
        (MEFIMAKE가 여기 있다)
```

### 정체성 선언

"MEFIMAKE는 크리에이티브 제작 단계를 가능한 빠르게 만들어, 인간이 실행 대신 전략에 시간을 쓸 수 있게 존재한다."

## 12. 마무리 반성

### 이 프로젝트가 나타내는 것

**실행에서 아키텍처로의 전환.**

예전에는 최적화했다: "태스크를 끝냈나?"
이제는 최적화한다: "태스크를 끝내는 시스템을 만들었나?"

**태스크에서 시스템으로의 전환.**

개별 태스크는 만족스럽지만 제한적이다.
시스템은 좌절스럽지만 확장 가능하다.

**것들을 만드는 것에서 플로우를 만드는 것으로의 전환.**

것을 만들기: 하나의 출력.
플로우를 만들기: 무한한 출력.

### 불편한 진실

시스템을 만드는 것은 기능을 만드는 것보다 느리다.

시스템을 설명하는 것은 기능을 설명하는 것보다 어렵다.

시스템에 대한 동의를 얻는 것은 기능보다 더 어렵다.

하지만 시스템은 복리로 증가한다. 기능은 그렇지 않다.

### 최종 생각

이 프로젝트는 임원들에게 AI를 설명하는 것을 멈추기 위한 도구로 시작했다.

워크플로우 자동화의 증명 포인트가 되었다.

내가 실제로 어떤 종류의 일을 하고 싶은지 드러냈다.

그리고 코드로, 디자인으로, 이 로그로 기록했다—실행자에서 아키텍트로의 전환을.

---

**Build high-converting creatives with AI.**

하지만 더 중요하게:

**것들을 만드는 시스템을 만들어라.**
