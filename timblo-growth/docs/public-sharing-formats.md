# 공개 공유 포맷

## GitHub

공유용 자산:
- **성장 로그** (`logs/YYYY-MM-DD.md`)
- **README** (철학, 리베뉴 루프, 로드맵)
- **자동화 아키텍처** (콘텐츠 재활용, 리드 흐름, 인수인계)

## LinkedIn

### 예시

**"오늘 실험한 것"**
> 오늘 실험: 인스타 DM 응답 시간을 2시간 이내로 맞춰봤어. 데모 요청 1건 들어왔다. 3명 팀으로 B2B SaaS 리베뉴 만드는 중.

**"3명으로 SaaS 리베뉴 구조 만드는 중"**
> 3명(총괄/영업/디자이너)이 리드→미팅→계약으로 가는 구조를 어떻게 설계하는지, 매일 조금씩 공개하고 있어. 채널 운영이 아니라 실제 자동화 사례 축적이 목표.

**"회의 AI 서비스 마케팅 실제 흐름"**
> 회의 AI B2B 서비스, 우리가 실제로 하고 있는 흐름: 수동 검증(0–2개월) → 하이브리드(2–4개월) → 반자동. 지금은 Phase 1, 어떤 메시지가 데모로 이어지는지 수집 중.

## Threads

### 스타일
- 짧은 한 문장 관찰
- 문제 제기
- 실제 숫자 (과장 없음)

### 예시

> 회의록 없는 회사 특징: "나중에 다시 공유할게"가 회의록이다.

> 상담 데이터 쌓이는데 매출 안 느는 이유 — 90%가 검색·분류에서 막힘.

> 3명으로 B2B 리드 만드는 중. 오늘 데모 1건.

## 블로그

### 포맷: 월별 케이스 스터디

1. 이번 달 목표
2. 한 것 (채널, 콘텐츠, 실험)

4. 다음 달에 바꿀 것
