WebScout Agent란?

WebScout Agent는 웹사이트의 모든 URL을 자동으로 발견하고 정리하여 Figma 디자인 임포트에 바로 사용할 수 있도록 준비해주는 AI 에이전트입니다.

  • 발견 — sitemap.xml 또는 지능형 크롤링으로 URL 자동 수집
  • 정규화 — 쿼리 제거, www/non-www 처리
  • 구조화 — 깊이 분석을 통한 웹사이트 계층 구조 파악
  • 내보내기 — Figma (html.to.design)에 바로 사용 가능한 깔끔한 URL 목록 생성

디자인 리빌드, 사이트 감사, Figma 워크플로우 자동화에 최적화되어 있습니다.

왜 만들었나요?

디자인을 리빌드할 때 URL을 하나씩 수동으로 입력하는 것은 비효율적이고 실수하기 쉽습니다. WebScout Agent는 이 전체 과정을 자동화하여 몇 시간 걸리던 수동 작업을 몇 초 만에 완료합니다.

에이전트 플로우

도메인 입력 ↓ Sitemap 발견 (robots.txt → sitemap.xml) ↓ 크롤링 대체 (sitemap이 없는 경우) ↓ URL 정규화 (www/non-www 병합, 쿼리 제거) ↓ 구조 분석 (깊이, 경로 계층 구조) ↓ 내보내기 (urls.txt / urls.csv)

자세한 플로우 다이어그램은 docs/agent-flow.md 참조

주요 기능

핵심 발견 기능

  • ✅ Sitemap 우선 접근 — sitemap.xml 자동 감지 및 파싱
  • ✅ 지능형 대체 — sitemap이 없을 경우 내부 링크 크롤링
  • ✅ 같은 출처 필터링 — 대상 도메인에서만 URL 수집
  • ✅ Robots.txt 파싱 — sitemap 위치 자동 발견

URL 처리

  • ✅ 쿼리 제거 — 쿼리 파라미터 선택적 제거
  • ✅ www/non-www 병합 — 동일 사이트로 처리
  • ✅ 해시 제거 — URL 프래그먼트 자동 정리
  • ✅ 경로 제외 — admin, API 등 경로 필터링

제어 및 제한

  • ✅ 깊이 제어 — 크롤링 깊이 제한 (기본값: 3)
  • ✅ 최대 URL 제한 — 무한 수집 방지 (기본값: 5000)
  • ✅ 요청 제한 — 요청 간 150ms 지연

대시보드 및 내보내기

  • ✅ 실시간 KPI 대시보드
  • ✅ 인터랙티브 URL 테이블 — 검색, 정렬, 필터
  • ✅ 내보내기 — urls.txt / urls.csv
  • ✅ 인사이트 탭 — Sitemap, 페이지 타입, Figma 준비도

WebScout Agent의 강점

비개발자 친화적

디자이너/마케터/PM이 코드 없이 사이트 전체 URL 맵을 만들고 내보낼 수 있습니다.

Figma 워크플로우 최적화

Figma, html.to.design 등에 바로 넣을 수 있는 형태로 URL을 정규화·내보내기 합니다.

운영 가능한 보안 수준

SSRF 방어, CORS allowlist, 레이트 리밋, 입력 검증, self-check 스크립트까지 포함합니다.

인사이트 지향 UI

Sitemap, 페이지 유형, Figma 준비도, 중요도 탭을 통해 리디자인 우선순위를 빠르게 결정합니다.

어떤 인사이트를 얻을 수 있나요?

  • 퍼널·구조 인사이트 — 랜딩 → 제품 → 전환 단계별 URL 분포, 깊이(Depth) 분석, 내부 링크 허브 페이지 파악
  • 콘텐츠·SEO 인사이트 — 카테고리별 URL 커버리지, 쿼리·파라미터 패턴, 중복·Thin 콘텐츠 후보
  • UX·전환 인사이트 — 전환 페이지 위치, 쿼리가 많은 URL, 구조적 병목 구간 후보
  • 디자인·리디자인 인사이트 — 템플릿/레이아웃 패밀리 파악, 중요도·준비도 기반 리디자인 우선순위 결정
시작하기

빠른 시작

필수 요구사항

  • Node.js 18+ (다운로드)
  • npm (Node.js와 함께 제공됨)

설치

# 저장소 클론
git clone https://github.com/yourusername/webscout-agent.git
cd webscout-agent

# 의존성 설치
npm install

Web UI 실행

npm start

브라우저에서 열기: http://localhost:3001

서버는 3001-3010 포트 중 사용 가능한 포트를 자동으로 찾습니다.

Web UI 사용법

  1. 도메인 입력 — 웹사이트 URL 입력 (예: https://example.com)
  2. 옵션 설정 — Depth, Max URLs, Query Strip, www/non-www, Exclude 경로
  3. "Collect URLs" 클릭 — 실시간 진행 상황 확인
  4. 결과 검토 — KPI 카드 및 URL 테이블 확인
  5. 내보내기 — urls.txt 또는 urls.csv 다운로드
  6. Figma로 임포트 — html.to.design을 사용하여 URL 임포트

"Use Demo Site" 버튼으로 https://webscout-demo.pages.dev에서 테스트할 수 있습니다.

CLI 사용법

# 기본 사용법
node collect-urls.mjs https://example.com

# 옵션과 함께
node collect-urls.mjs https://example.com --depth=4 --max=8000

CLI 옵션

  • --depth=N — 최대 크롤링 깊이 (기본값: 3)
  • --max=N — 수집할 최대 URL 수 (기본값: 5000)

결과 검증 (PowerShell)

.\verify-urls.ps1 -TargetDomain "https://example.com"

검증 항목: 같은 출처 필터링, 해시 제거, 쿼리 문자열 처리, www/non-www 정규화

문제 해결

포트가 이미 사용 중

서버는 자동으로 3001-3010 포트를 시도합니다. 모두 사용 중이면:

# 포트 3001의 프로세스 종료 (Windows PowerShell)
netstat -ano | findstr :3001
taskkill /PID <PID> /F

URL이 수집되지 않음

  • Sitemap 확인 — https://yourdomain.com/sitemap.xml 방문
  • SPA 사이트 — JavaScript 렌더링이 필요할 수 있음 (아직 지원 안 됨)
  • 봇 차단 — 일부 사이트는 자동화된 요청을 차단함

PowerShell 스크립트가 실행되지 않음

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\verify-urls.ps1 -TargetDomain "https://example.com"
