# 비전 및 로드맵

## 장기 비전

**MEFIMAKE는 퍼포먼스 크리에이티브 제작의 운영 체제가 될 것입니다.**

단순한 디자인 도구가 아닌, 중심 허브가 될 것입니다:
- 크리에이티브 브리프가 변형으로 전환
- AI가 성과 데이터 기반 최적화 제안
- 팀이 크리에이티브 전략에 협업
- 광고가 플랫폼에 직접 배포

---

## 전략적 기둥

### 1. 속도
모든 기능은 다음으로 평가됩니다: "이것이 크리에이티브 제작을 더 빠르게 만드는가?"

### 2. 지능
카피 생성부터 레이아웃 제안까지 모든 단계에서 AI 지원 보조.

### 3. 통합
마케터가 이미 사용하는 도구와 플랫폼에 원활한 연결.

### 4. 단순함
전문적인 복잡성 없이 전문적인 역량.

## 기술 로드맵

### 현재 아키텍처
```
브라우저 (Vanilla JS)
    └── Local Storage
```

### 2단계 아키텍처
```
브라우저 (Vanilla JS)
    └── REST API
        └── 데이터베이스 (PostgreSQL)
        └── AI 서비스 (OpenAI/Anthropic)
        └── 스토리지 (S3/Cloudflare R2)
```

### 4단계 아키텍처
```
브라우저 (React/Vue)
    └── GraphQL API
        └── 인증 서비스 (Auth0/Clerk)
        └── 데이터베이스 (PostgreSQL)
        └── AI 서비스
        └── 스토리지 (S3)
        └── 큐 (Redis)
        └── 분석 (ClickHouse)
```

## 비목표

의도적으로 만들지 않을 것들:

1. **풀 디자인 도구** - 일반 디자인에서 Figma와 경쟁하지 않음
2. **비디오 에디터** - 간단한 모션은 지원하지만 풀 비디오 편집은 아님
3. **소셜 미디어 스케줄러** - 퍼블리싱은 범위 밖
4. **분석 플랫폼** - 분석 도구를 대체하지 않고 통합

## 우선순위 결정 방법

모든 기능 요청은 다음으로 평가됩니다:

1. **임팩트** - 얼마나 많은 사용자가 혜택을 받는가? 얼마나 많은 시간이 절약되는가?
2. **노력** - 필요한 엔지니어링 시간
3. **전략적 적합성** - 비전과 일치하는가?
4. **수익 잠재력** - 수익화를 가능하게 하는가?

우선순위 = (임팩트 × 전략적 적합성) / 노력

## 열린 질문들

아직 탐색 중인 결정들:

1. **가격 모델** - 프리미엄 vs. 구독 vs. 사용량 기반?
2. **프레임워크 선택** - Vanilla JS 유지 또는 React/Vue로 마이그레이션?
3. **AI 제공자** - OpenAI vs. Anthropic vs. 셀프 호스팅?
4. **타겟 시장** - SMB 집중 또는 엔터프라이즈 우선?

## 로드맵 기여 방법

로드맵에 대한 의견을 환영합니다:

1. **GitHub Issues** - `enhancement` 라벨로 기능 요청
2. **Discussions** - 방향에 대한 열린 대화
3. **유저 리서치** - 워크플로우와 페인 포인트 공유

최고의 기능은 실제 사용자 문제에서 나옵니다.
