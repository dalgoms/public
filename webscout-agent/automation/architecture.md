# 자동화 아키텍처

## 개요

자동화는 **Phase 2**부터 시작합니다. Phase 1은 메시지 검증과 데모 유도 요인 학습을 위해 완전 수동입니다.

---

## 1. 콘텐츠 재활용 흐름

**내용**: 원본 콘텐츠 1개 → 여러 채널 포맷

```
원본 콘텐츠 (예: 인스타 포스트)
    ↓
LinkedIn: 요약 버전
    ↓
Threads: 한 문장·질문 추출
    ↓
블로그: 장문 초안 생성
```

**도구**: Make.com 등. 새 콘텐츠 시 트리거; 채널별 포맷으로 출력.

---

## 2. 리드 인테이크 흐름

**내용**: 모든 리드 소스 → 단일 DB → 영업 알림

```
DM (인스타) / 폼 (웹사이트)
    ↓
Webhook
    ↓
Make.com
    ↓
Notion DB 또는 Supabase
    ↓
Slack 알림 → 영업
```

**규칙**:
- 영업은 24시간 내 응답
- 각 리드 태깅: 산업, 사용 사례, 긴급도

---

## 3. 태깅

| 태그 | 용도 |
|------|------|
| `industry` | 섹터별 세그먼트 (공공, 금융, 대기업 등) |
| `usecase` | 회의 유형, 컴플라이언스 요구사항 |
| `urgency` | Hot / Warm / 육성 |

활용:
- 영업 후속 우선순위
- 콘텐츠 개인화
- Phase 3 스코어링

---

## 4. 영업 인수인계

| 점수 | 액션 |
|------|------|
| 높음 | 48시간 내 미팅 |
| 중간 | 육성 시퀀스 |
| 낮음 | 뉴스레터 / 장기 육성 |

**Phase 3**에서 자동 스코어링 추가 (기업 규모, 응답 속도, 니즈 표현 등).

---

## 5. Phase별 자동화

| Phase | 자동화 | 수동 |
|-------|--------|------|
| Phase 1 | 없음 | 모든 콘텐츠, 모든 리드 처리 |
| Phase 2 | 리드 인테이크, 태깅, Slack 알림, 콘텐츠 재활용 | 콘텐츠 제작, 영업 통화 |
| Phase 3 | 스코어링, 육성, 콘텐츠 초안 | 전략, 고터치 영업 |
