# 배포 가이드

이 가이드는 Vercel을 사용하여 MEFIMAKE를 프로덕션에 배포하는 방법을 다룹니다.

---

## 개요

MEFIMAKE는 서버 측 요구 사항이 없는 정적 웹 애플리케이션입니다. 배포는 간단합니다:

1. 코드를 GitHub에 푸시
2. Vercel에 연결
3. 배포

---

## 사전 요구 사항

- GitHub 계정
- Vercel 계정 (무료 티어 사용 가능)
- 로컬에 Git 설치
- Node.js 16+ (선택 사항, 로컬 테스트용)

---

## GitHub 설정

### 1. 레포지토리 생성

```bash
# 프로젝트 디렉토리로 이동
cd meta-ad-generator

# git 초기화 (아직 안 되어 있다면)
git init

# 모든 파일 추가
git add .

# 초기 커밋 생성
git commit -m "Initial commit: MEFIMAKE v1.0.0"

# 원격 origin 추가
git remote add origin https://github.com/YOUR_USERNAME/mefimake.git

# GitHub에 푸시
git push -u origin main
```

### 2. 레포지토리 설정

권장 설정:
- **브랜치 보호:** main 브랜치에 PR 필요
- **가시성:** Public (오픈 소스용) 또는 Private
- **토픽:** `ad-creative`, `design-tool`, `javascript`, `no-build`

### 3. .gitignore

```gitignore
# OS 파일
.DS_Store
Thumbs.db

# IDE
.vscode/
.idea/

# 의존성 (있는 경우)
node_modules/

# 로컬 파일
*.local
.env

# 빌드 출력 (이 프로젝트에는 해당 없음)
dist/
build/
```

## Vercel 배포

### 1. 레포지토리 연결

1. [vercel.com](https://vercel.com)으로 이동
2. GitHub으로 로그인
3. "Add New Project" 클릭
4. "Import Git Repository" 선택
5. `mefimake` 레포지토리 선택
6. "Import" 클릭

### 2. 빌드 설정 구성

MEFIMAKE는 빌드 단계가 없으므로:

| 설정 | 값 |
|------|------|
| Framework Preset | Other |
| Build Command | (비워둠) |
| Output Directory | `.` |
| Install Command | (비워둠) |

### 3. 환경 변수

기본 배포에는 환경 변수가 필요 없습니다.

향후 기능(API 키 등)을 위해:

1. Project Settings → Environment Variables로 이동
2. 필요에 따라 변수 추가
3. 재배포하여 적용

### 4. 배포

"Deploy"를 클릭하고 완료를 기다립니다 (~30초).

사이트는 다음에서 사용 가능합니다:
- `https://your-project.vercel.app`
- `https://your-project-git-main-username.vercel.app`

## 커스텀 도메인

### 1. 도메인 추가

1. Project Settings → Domains로 이동
2. 도메인 입력 (예: `mefimake.com`)
3. "Add" 클릭

### 2. DNS 구성

Vercel이 추가할 DNS 레코드를 제공합니다:

**옵션 A: Apex 도메인**
```
Type: A
Name: @
Value: 76.76.21.21
```

**옵션 B: 서브도메인**
```
Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

### 3. SSL 인증서

Vercel이 Let's Encrypt를 통해 자동으로 SSL 인증서를 프로비저닝합니다. 추가 작업 필요 없음.

## 프로덕션 최적화

### 1. vercel.json 구성

프로젝트 루트에 `vercel.json` 생성:

```json
{
    "headers": [
        {
            "source": "/assets/(.*)",
            "headers": [
                {
                    "key": "Cache-Control",
                    "value": "public, max-age=31536000, immutable"
                }
            ]
        },
        {
            "source": "/(.*\\.css)",
            "headers": [
                {
                    "key": "Cache-Control",
                    "value": "public, max-age=31536000, immutable"
                }
            ]
        },
        {
            "source": "/(.*\\.js)",
            "headers": [
                {
                    "key": "Cache-Control",
                    "value": "public, max-age=31536000, immutable"
                }
            ]
        }
    ],
    "rewrites": [
        {
            "source": "/(.*)",
            "destination": "/index.html"
        }
    ]
}
```

### 2. 보안 헤더

보안 헤더 추가:

```json
{
    "headers": [
        {
            "source": "/(.*)",
            "headers": [
                {
                    "key": "X-Content-Type-Options",
                    "value": "nosniff"
                },
                {
                    "key": "X-Frame-Options",
                    "value": "DENY"
                },
                {
                    "key": "X-XSS-Protection",
                    "value": "1; mode=block"
                },
                {
                    "key": "Referrer-Policy",
                    "value": "strict-origin-when-cross-origin"
                }
            ]
        }
    ]
}
```

### 3. 압축

Vercel이 자동으로 Brotli/Gzip 압축을 적용합니다. 구성 필요 없음.

## 프리뷰 배포

모든 풀 리퀘스트는 자동으로 프리뷰 배포를 받습니다:

1. GitHub에서 PR 생성
2. Vercel 봇이 PR 코멘트에 프리뷰 URL 게시
3. 프리뷰 URL에서 변경 사항 검토
4. PR 머지하여 프로덕션에 배포

### 프리뷰 URL 형식
```
https://project-name-git-branch-name-username.vercel.app
```

## 지속적 배포

### 자동 배포

기본적으로 `main`에 모든 푸시가 프로덕션 배포를 트리거합니다.

### 배포 훅

외부 트리거(CI/CD, Slack 등)를 위해:

1. Project Settings → Git으로 이동
2. "Deploy Hooks"로 스크롤
3. 새 훅 생성
4. 웹훅 URL을 사용하여 배포 트리거:

```bash
curl -X POST https://api.vercel.com/v1/integrations/deploy/prj_xxxxx/xxxxxxx
```

## 롤백

### Vercel 대시보드를 통해

1. Project → Deployments로 이동
2. 작동하는 배포 찾기
3. "..." 메뉴 클릭
4. "Promote to Production" 선택

### CLI를 통해

```bash
# Vercel CLI 설치
npm i -g vercel

# 배포 목록
vercel ls

# 특정 배포 프로모션
vercel promote [deployment-url]
```

## 문제 해결

### 빌드 실패

**문제:** Vercel이 프로젝트를 빌드할 수 없음.

**솔루션:** 빌드 커맨드가 비어 있는지 확인 (빌드 필요 없음).

### 새로고침 시 404

**문제:** 직접 URL 접근 시 404 반환.

**솔루션:** `vercel.json`에 리라이트 규칙 추가:
```json
{
    "rewrites": [
        { "source": "/(.*)", "destination": "/index.html" }
    ]
}
```

### 에셋이 로드되지 않음

**문제:** 프로덕션에서 이미지나 폰트가 로드되지 않음.

**솔루션:** 경로가 상대적인지 확인 (`assets/logo.png`이 아닌 `/assets/logo.png`).
